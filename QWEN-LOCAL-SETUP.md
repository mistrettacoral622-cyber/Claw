# 配置本地 Qwen3.5-9B 服务

## 问题修复

已修复 Qwen3.5-9B 模型的工具调用循环问题。

**问题原因**：Qwen3.5-9B 是一个小模型，会误解系统的 dispatch hints（工具调用提示），将其当作工具调用指令，导致进入无限循环。

**解决方案**：
- 添加了 `SMALL_MODEL_PATTERNS` 列表来识别小模型
- 对于小模型，系统会跳过所有 dispatch hints，避免误解
- **重要**：Qwen3.5-9B 仍然是多模态模型，支持图像理解功能

**修改文件**：
- `shared/chat-dispatch-hints.ts`：添加小模型检测逻辑
- `src/stores/chat.ts`：传递当前模型信息给 dispatch hints 函数

## 在应用中配置步骤

### 方案 1：使用 Custom 提供商（推荐）

1. **打开应用设置**
   - 点击右上角的设置图标
   - 或使用快捷键打开设置

2. **添加 Custom 提供商**
   - 找到 "Providers" 或"提供商"部分
   - 点击 "Add Provider" 或 "添加提供商"
   - 选择 "Custom" 类型

3. **填写配置信息**
   ```
   Provider Type: Custom
   Name: Qwen3.5-9B Local (可自定义名称)
   Base URL: http://10.101.80.18:8888/v1
   Model ID: Qwen3.5-9B
   API Key: any-key (随意填写，不校验)
   ```

4. **保存并启用**
   - 点击保存
   - 确保提供商已启用（enabled）

5. **选择模型**
   - 在聊天界面右下角的模型选择器中
   - 选择你刚配置的 "Qwen3.5-9B Local"

### 方案 2：使用 Ollama 提供商（如果你的服务兼容 Ollama API）

如果你的服务实现了 Ollama 兼容的 API：

1. **选择 Ollama 提供商**
   - 在设置中找到 Ollama
   - 或添加新的 Ollama 账户

2. **配置**
   ```
   Base URL: http://10.101.80.18:8888/v1
   Model ID: Qwen3.5-9B
   API Key: 不需要（Ollama 默认不需要）
   ```

## 验证配置

配置完成后，测试一下：

1. **发送简单消息**
   ```
   你好，请介绍一下你自己
   ```

2. **检查响应**
   - 应该能正常收到回复
   - 不会卡在 "工具调用处理中..."
   - 底部状态栏应显示 "Qwen3.5-9B"

## 注意事项

### ✅ 适用场景
- 纯文本对话
- 代码生成和解释
- 文档编写
- 简单的问答

### ⚠️ 限制
由于 Qwen3.5-9B 被配置为纯文本模型，以下功能可能受限：
- **工具调用**：不会主动调用工具（避免循环问题）
- **图像理解**：不支持图像输入
- **复杂任务**：多步骤任务可能需要手动引导

### 💡 建议
如果需要工具调用和复杂任务处理，建议：
1. 配置一个更大的模型作为 fallback
2. 或者使用其他支持工具调用的模型（如 DeepSeek、GLM-4 等）

## 故障排查

### 问题 1：连接失败
**症状**：提示无法连接到服务器

**解决**：
- 检查 URL 是否正确：`http://10.101.80.18:8888/v1`
- 确认服务是否在运行
- 检查防火墙设置
- 尝试在浏览器访问：`http://10.101.80.18:8888/v1/models`

### 问题 2：认证失败
**症状**：提示 API Key 无效

**解决**：
- 如果服务不校验 API Key，随意填写即可（如 "sk-test"）
- 如果服务需要特定格式，按要求填写

### 问题 3：模型 ID 不匹配
**症状**：提示模型不存在

**解决**：
- 确认模型 ID 大小写：`Qwen3.5-9B` vs `qwen3.5-9b`
- 尝试访问 `/v1/models` 端点查看可用模型列表
- 根据实际返回的模型 ID 调整配置

### 问题 4：响应格式错误
**症状**：收到错误或乱码

**解决**：
- 确认服务实现了 OpenAI 兼容的 API
- 检查 API Protocol 设置（应该是 `openai-completions`）
- 查看开发者工具控制台的错误信息
